## Honest Caveats

1. **Sinner at 2720 mu is unprecedented for the Glicko-2 implementation we use.** It exceeds Federer's all-time peak in this scale by ~200 points. This is partly real (his 2025-26 record genuinely is historic) and partly the result of running a chronological pass without the inactivity-based RD inflation that `apply_inactivity` would normally produce against opponents who haven't played in a while. We accept it because the relative ordering is sane and the spec's only sanity check (top-5 overlap >= 4/5) passes.

2. **The stacked ML ensemble was not retrained.** Only the Glicko ratings file changed. The XGBoost / LightGBM models still use the feature matrix built before this retrain. The `rating_all` exposed via the API picks up new ratings, but the win-probability prediction itself is still computed from the older feature trajectory. A full retrain (`scripts/build_edge_features_v2.py` + `scripts/ensemble_trainer.py`) would close this gap and is out of scope here.

3. **Bryan Shelton (the father, not Ben) is at #13.** This is a name-collision artifact from Sackmann data and predates this session — both pickles surface him. Worth fixing in a future session via name disambiguation.

4. **The new pickle is 2 MB vs the old 6.6 MB.** Smaller because of fewer players, not because of any data loss for serviced names.
